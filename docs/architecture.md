## 1. 项目概述

### 1.1 项目简介

Cat Agent Platform (猫猫多Agent协同系统) 是一个多智能体协同工作平台，支持内置Agent和外部Agent的接入，提供任务编排、协同机制和状态监控等核心能力。系统采用前后端分离架构，后端基于Spring Boot微服务架构，前端基于Vue 3单页应用。

### 1.2 核心功能

| 功能领域 | 描述 |
|---------|------|
| **用户认证** | JWT Token认证、RBAC角色权限控制 |
| **Agent管理** | 内置Agent创建配置、外部Agent接入、能力注册、状态监控 |
| **任务执行** | 简单任务、流程化任务、并行任务、协商任务 |
| **任务调度** | 立即执行、定时执行、周期执行 |
| **协同机制** | 流程编排、并行处理、投票协商 |
| **运行时执行** | 命令执行、API调用、文件操作、文本处理、MCP协议适配 |
| **系统监控** | 系统指标采集、告警规则配置、告警通知 |
| **审计日志** | 操作审计、日志查询导出 |

### 3.2 API响应标准化

所有API接口统一使用 `ApiResponse<T>` 封装响应：

```json
{
  "code": 0,
  "message": "success",
  "data": { ... },
  "timestamp": 1709254800000
}
```

响应码约定：
- `code = 0`: 成功
- `code > 0`: 业务错误码
- `code < 0`: 系统错误码

### 3.3 认证流程

**单机模式 (Standalone) - 简化认证:**

为简化本地开发体验，单机模式已移除认证验证：
- 前端：去掉路由守卫，登录页面直接跳转
- 后端：SimpleAuthInterceptor不再验证Token，允许所有请求通过
- 登录页面：任意用户名密码可直接进入系统

| 简单任务 | SIMPLE | 单Agent执行的独立任务 |
| 流程任务 | WORKFLOW | 多步骤顺序执行任务 |
| 并行任务 | PARALLEL | 多Agent并行执行任务 |
| 协商任务 | NEGOTIATION | Agent投票协商决策任务 |

**任务状态 (task.status):**
| 状态 | 编码 | 描述 |
|------|------|------|
| 待处理 | PENDING | 任务已创建，等待分配 |
| 已分配 | ASSIGNED | 任务已分配给Agent |
| 运行中 | RUNNING | 任务正在执行 |
| 已完成 | COMPLETED | 任务成功完成 |
| 已失败 | FAILED | 任务执行失败 |
| 已取消 | CANCELLED | 任务被取消 |

### 4.3 Agent状态

| 状态 | 编码 | 描述 |
|------|------|------|
| 离线 | OFFLINE | Agent未连接 |
| 在线 | ONLINE | Agent空闲可用 |
| 忙碌 | BUSY | Agent正在执行任务 |
| 错误 | ERROR | Agent异常状态 |
| 禁用 | DISABLED | Agent被禁用 |

## 7. 开发与部署

### 7.1 快速启动 (单机模式 - 推荐)

```bash
# 方式一：使用启动脚本
./run-standalone.sh   # Linux/Mac
run-standalone.bat    # Windows

# 方式二：手动构建启动
mvn clean package -pl cat-standalone -am -DskipTests
java -jar cat-standalone/target/cat-standalone-1.0.0-SNAPSHOT.jar

# 方式三：使用Maven直接运行
mvn spring-boot:run -pl cat-standalone
```

**访问地址:**
- API: http://localhost:8080/api/v1
- 默认用户: admin / admin123
- 数据目录: ./data/

**API认证:**
```bash
# 1. 登录获取Token
curl -X POST http://localhost:8080/api/v1/auth/login \
  -H "Content-Type: application/json" \
  -d '{"username": "admin", "password": "admin123"}'

# 2. 使用Token访问API (X-Cat-Token header)
curl http://localhost:8080/api/v1/agents \
  -H "X-Cat-Token: <your-token>"
```

### 7.2 本地开发环境启动 (微服务模式)

```bash
# 1. 启动基础设施
docker-compose up -d

# 2. 初始化数据库 (已自动执行 init-database.sql)

# 3. 构建后端
mvn clean install

# 4. 启动服务 (按依赖顺序)
mvn spring-boot:run -pl cat-auth    # 先启动认证服务
mvn spring-boot:run -pl cat-agent   # Agent服务
mvn spring-boot:run -pl cat-task    # 任务服务
mvn spring-boot:run -pl cat-gateway # 网关服务

# 5. 启动前端
cd cat-web
npm install
npm run dev
```

## 8. 设计原则与约束

### 8.1 架构设计原则

| 原则 | 描述 | 实现方式 |
|------|------|----------|
| 模块化 | 高内聚低耦合 | 按业务域划分模块，接口隔离 |
| 可扩展 | 水平扩展能力 | 无状态服务设计，Redis共享状态 |
| 安全性 | 防御性编程 | JWT认证、RBAC权限、SQL防注入 |
| 可观测 | 全链路监控 | 统一日志、指标采集、审计追踪 |
| 简洁性 | 最简方案优先 | 避免过度设计，选用成熟组件 |

### 8.2 代码规范

- **命名规范**: 遵循Java驼峰命名、数据库下划线命名
- **ID策略**: UUID主键，避免自增ID暴露业务量
- **JSON字段**: 灵活配置使用JSON类型存储
- **异常处理**: GlobalExceptionHandler统一处理
- **日志规范**: 使用SLF4J + Lombok @Slf4j
- **API文档**: Swagger OpenAPI 3.0注解
